1 A pipeline starts with…

<pipeline ssi-v="1">

2 Let’s load some components

We load the components, which we need to to ensemble our pipeline. Components can be imported from dynamically-linked libraries (.dll on Windows or .so on Linux), which we call plug-ins.

<register>
    <load name="mouse"/>
    <load name="graphic"/>        
</register>

3 Connect a sensor

We add a mouse sensor, which offers two streams: the cursor position and the state of the button. We connect to both streams and assign unique pins, here button and cursor. Later we can use these pins to connect the streams with a component. To create the component, we need to know its name. A useful source of information is therefore the API of SSI. It features a list of all dll files in the left menu. When you click on one, you’ll get a list of the embedded components. If you select a component you will see its name, type and options. In case of Mouse you will learn that the component is a sensor and that an option mask exists. It determines which buttons will fire when clicked by the user. We can overwrite the default value by setting mask="1". Now, as long as the left mouse button is pressed the cursor stream will be 1.

<sensor create="Mouse" mask="1">
    <output channel="button" pin="button" />
    <output channel="cursor" pin="cursor" />
</sensor>

4 Add some visualization

As mentioned before we can connect to streams by using their pins. Here, we add two graphs to visualize the cursor and button streams during the last 10 seconds. This time, we also assign an id to the instances by adding :<id> after a component’s name. In fact, we assign the id plot twice. However, SSI automatically adds a trailing number to keep the two instances apart.

<consumer create="SignalPainter:plot" title="CURSOR" size="10.0">
    <input pin="cursor" frame="0.2s" />        
</consumer>
<consumer create="SignalPainter:plot" title="BUTTON" size="10.0">
    <input pin="button" frame="0.2s" />        
</consumer>

5 Align the windows

By default all windows will open in the same location on the screen. To move them around an instance of Decorator is added. We refer to instances we want to reposition by their id. In this case, the console (which always has the id console) and the two graphs we just added. By using the wildcard symbol * we refer to both plots and let SSI assemble the two windows in the mentioned area.

<object create="Decorator" icon="true" title="Pipeline">
    <area pos="0,0,400,600">console</area>
    <area pos="400,0,400,600">plot*</area>
</object>

6 And finally close the pipeline.

To run the pipeline double click 01_hello_world.cmd.

</pipeline>
